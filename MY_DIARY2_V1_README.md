# マイ日記帳2（IndexedDB対応版）

## 📝 概要
**マイ日記帳2**は、マイ日記帳1と全く同じ機能を持つ独立した日記アプリです。
本バージョンより **IndexedDB** を採用し、ストレージ容量の制限を大幅に緩和しました。

---

## 🆕 IndexedDB 対応（最新変更点）

| 項目 | 旧バージョン | 新バージョン |
|------|------------|------------|
| ストレージ | localStorage（上限 約5MB） | IndexedDB（数百MB〜1GB以上） |
| 起動画面 | なし | ローディング画面あり |
| 保存通知 | なし | 右下に「✅ 保存しました」表示 |
| 既存データ | 手動移行が必要 | 初回起動時に**自動移行** |

### IndexedDB 仕様
| 項目 | 内容 |
|------|------|
| DB名 | `diary2db` |
| ストア名 | `appdata` |
| 保存キー | `diary2_articles` / `diary2_profile` / `diary2_custom_links` |
| 容量上限 | ブラウザの空き容量次第（通常数百MB〜1GB以上） |

---

## ✨ 特徴

### 完全独立動作
- **データ分離**: マイ日記帳1とデータが完全に分離
- **IndexedDB キー**:
  - `diary2_articles`（記事データ）
  - `diary2_profile`（プロフィールデータ）
  - `diary2_custom_links`（カスタムリンク設定）

### 同じ機能
1. **ダッシュボード**
   - 年タブ（2026/2025/2024...）
   - 12ヶ月ボックス（6列×2行）
   - 各月1-5日表示、もっと見るで全日付展開

2. **記事機能**
   - ブロックエディタ（テキスト/画像を自由配置）
   - 下書き保存機能
   - 複数画像アップロード
   - タグ・カテゴリー
   - コメント機能

3. **プロフィール**
   - 写真、名前、誕生日、血液型、星座、身長
   - カスタム項目（複数追加可能）

4. **カスタムリンク**
   - ヘッダーに最大2つのリンクボタン
   - タイトルとURLを編集可能

5. **検索・エクスポート/インポート**
   - 全文検索
   - JSONエクスポート/インポート（IndexedDB保存も自動実行）

## 🚀 GitHub公開方法

### ステップ1: リポジトリ作成
1. GitHubで新しいリポジトリ作成
2. リポジトリ名: `my-diary2`（または任意の名前）
3. Public を選択

1. `my-diary2_idb.html` を `index.html` にリネーム

1. Settings → Pages
2. Source: `main` branch → Root → Save

### ステップ4: アクセス
数分後、以下のURLでアクセス可能：
```
https://あなたのユーザー名.github.io/my-diary2/
```

## 💾 データ管理

### エクスポート
1. 「エクスポート」ボタンをクリック
2. `diary_backup_YYYY-MM-DD.json` がダウンロードされる

### インポート
1. 「インポート」ボタンをクリック
2. バックアップJSONファイルを選択
3. データが IndexedDB に復元される

### 注意事項
- マイ日記帳1（`diarydb`）とマイ日記帳2（`diary2db`）は完全に独立した IndexedDB を使用
- 容量はそれぞれ独立して確保される
- ブラウザのキャッシュクリアではデータは消えない（IndexedDB は永続ストレージ）
- **定期的にJSONエクスポートでバックアップを取ることを推奨**

## 🆘 よくある質問

**Q: マイ日記帳1とデータは共有されますか？**
A: いいえ。`diarydb`（日記帳1）と `diary2db`（日記帳2）は完全に独立したIndexedDBです。

**Q: IndexedDBの容量はどれくらいですか？**
A: ブラウザの空き容量に依存しますが、通常数百MB〜1GB以上使えます。localStorage（5MB）と比べ大幅に拡張されています。

**Q: データが消えることはありますか？**
A: IndexedDB はブラウザのキャッシュクリアでは消えません。ただし「サイトデータを削除」した場合は消える可能性があります。定期的にエクスポートしてください。

**Q: スマホでも使えますか？**
A: はい、レスポンシブデザインでスマホでも使えます。IndexedDBもスマホブラウザで対応しています。
